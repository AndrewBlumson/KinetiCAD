Phase 1: WebGPU Canvas + OpenCascade.js Worker
This phase wires up the 3D rendering and the CAD kernel. By the end of this phase, a 10mm cube generated by OpenCascade.js is rendered in the central canvas with PBR lighting, orbit camera, grid floor, and origin axes.
Dependencies to install (pin these exact versions, do not upgrade):

three@0.184.0
@types/three@0.184.0
opencascade.js@2.0.0-beta.94e2944
comlink@4.4.1

File structure to create:
src/
  three/
    Scene.tsx              React component, mounts canvas, owns the WebGPU renderer and render loop
    sceneSetup.ts          createScene, createCamera, createLights, createGrid, createAxes
    OrbitControls.ts       Standard Three.js OrbitControls wrapper
  cad/
    cadWorker.ts            Web Worker entry, Comlink-exposed kernel API
    cadClient.ts            Main-thread Comlink client, typed wrapper around the worker
    types.ts                Shared types between worker and main thread (TessellatedMesh, etc)
public/
  opencascade.wasm          OpenCascade WASM binary, copied at install time or via Vite asset handling
WebGPU canvas requirements:

Use Three.js WebGPU renderer (import { WebGPURenderer } from 'three/webgpu')
Detect WebGPU support on mount. If unavailable, render a centred message: "KinetiCAD requires WebGPU. Open in Chrome on an M-series Mac." and stop. No WebGL2 fallback.
Background colour #0A0E1A
Grid floor: 200mm x 200mm extent, 10mm cell, fades with distance via custom shader or GridHelper with opacity-based depth fade. Grid colour #141B2E.
Origin axes: X red #FF4444, Y green #44FF44, Z blue #4488FF. Length 50mm. Slightly thicker than default AxesHelper lines, achieved with Line2 from examples or just LineSegments with custom material.
PBR environment lighting via RoomEnvironment from three/examples/jsm/environments/RoomEnvironment.js, processed through PMREMGenerator and assigned to scene.environment.
One directional key light at intensity 0.6, positioned at (50, 80, 30), casting soft shadows.
OrbitControls with damping enabled, target at origin, default position (80, 60, 80) looking back at origin. Min zoom 20mm, max zoom 500mm.
Render loop using renderer.setAnimationLoop, not requestAnimationFrame. Required for WebGPU.
Resize handler that updates camera aspect and renderer size on window resize.
FPS counter logged to console once per second in dev mode only.

OpenCascade.js worker requirements:

Worker file at src/cad/cadWorker.ts, imported in main thread with Vite's ?worker suffix: import CadWorker from './cadWorker?worker'.
Worker imports opencascade.js and initialises the WASM module on startup. WASM binary served from /opencascade.wasm (use Vite's asset handling or copy to public/).
Kernel API exposed via Comlink with the following methods for this phase:

init(): Promise<{ initTimeMs: number; version: string }> resolves when kernel is ready
createTestCube(sizeMm: number): Promise<TessellatedMesh> returns positions, normals, indices


TessellatedMesh shape:

ts  type TessellatedMesh = {
    positions: Float32Array; // xyz triplets
    normals: Float32Array;   // xyz triplets
    indices: Uint32Array;    // triangle indices
  };

Main-thread client at src/cad/cadClient.ts wraps the Comlink proxy with a typed singleton: getCadKernel() returns a memoised initialised kernel.

Test cube rendering:

On Modeller view mount, after kernel init resolves, call createTestCube(10).
Convert the TessellatedMesh to a BufferGeometry, attach to a Mesh with MeshStandardMaterial (colour #A0A8B5, metalness 0.4, roughness 0.5).
Add to scene at origin.
Cube should be visibly lit, casting subtle shadows, orbit camera should rotate around it.

Loading state:

Before kernel init resolves, render a centred overlay on the canvas with "Loading CAD kernel..." in JetBrains Mono, Replit Orange, with a subtle pulsing animation.
Kernel init time logged to console as [KERNEL] Ready in 4231ms or similar.

Acceptance criteria:

Deployed URL opened in external Chrome on an M-series Mac shows: dark navy background, faded grid floor, origin axes, a single 10mm cube at origin with PBR shading, orbit controls working.
Kernel init time logged to console under 15 seconds on first load.
FPS holds 60 with the cube rendering.
WebGPU detection works: forcing WebGL-only browsers shows the fallback message.
