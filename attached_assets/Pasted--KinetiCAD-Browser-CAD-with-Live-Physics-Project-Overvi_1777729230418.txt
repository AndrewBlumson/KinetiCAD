# KinetiCAD: Browser CAD with Live Physics

## Project Overview

**Application Name:** KinetiCAD (placeholder, change if preferred)
**Built By:** Adevious AI Ltd (Andrew Blumson)
**Event:** Replit 10 Buildathon, 02/05/2026, 13:00 BST start, 24-hour Free Agent window
**Purpose:** Browser-based parametric CAD tool where users design mechanical parts, assemble them with mate joints, then press play to watch the assembly physically simulate under gravity and joint motors. Differentiator: every other browser CAD tool stops at static geometry; this one moves. Build a four-bar linkage, press play, watch it cycle. Build a piston-crank, press play, watch it pump.
**Target Users:** Showcase audience on LinkedIn and X. Engineers, makers, Replit community, Amjad Masad.
**Reference Inspirations:** Onshape (cloud CAD UX), Fusion 360 (mate joints), Rapier3D demos (physics feel), Onshape Drive (assembly play mode).

## Technology Stack

**Frontend:**
- React 19.x with TypeScript 6.x (strict mode)
- Vite 8.x build tool
- Tailwind CSS 3.4.x for utility styling
- Zustand for application state (sketch tree, feature tree, assembly state, simulation state)
- React.lazy for code splitting between Modeller and Simulator views

**3D Rendering:**
- Three.js r184 with WebGPU renderer (canonical for new showcase builds)
- No WebGL2 fallback. Target M4/M5 MacBook 24GB Chrome only.
- PBR materials with environment lighting (HDR pmrem)
- Custom selection outline shader, Replit Orange #FF6B1A
- Grid floor with origin axes, fading at distance

**CAD Kernel:**
- OpenCascade.js (full B-rep CAD kernel compiled to WASM)
- Runs in dedicated Web Worker to keep main thread responsive
- Worker communication via Comlink for typed RPC
- Exports tessellated meshes to main thread for Three.js rendering

**Physics:**
- Rapier3D (Rust to WASM, browser-native physics engine)
- Joints: fixed, revolute (pin), prismatic (slider), spherical (ball), planar (slot)
- Joint motors for actuation
- Gravity, mass derived from material density and computed volume

**Persistence:**
- Browser localStorage for save/load
- JSON serialisation of sketch tree, feature tree, assembly graph
- No backend required for v1

**Deployment:**
- Replit Autoscale Deployment
- Test on deployed URL in external M4/M5 Chrome (Replit preview iframe does not support WebGPU)

**Currency:** GBP (£). DD/MM/YYYY dates. HH:MM:SS times. en-GB locale. UK English spelling everywhere.

## Critical Rules

1. **WebGPU only.** No WebGL2 fallback. If WebGPU is missing show an "Open in Chrome on M-series Mac" message.
2. **OpenCascade.js runs in a Web Worker.** Never touch the kernel from the main thread.
3. **Test on the deployed URL externally.** Replit preview iframe will not run WebGPU.
4. **One prompt per phase.** STOP gates between phases. Andrew reviews then unlocks the next phase.
5. **Commit after every phase.** Never push until Andrew explicitly says push.
6. **No dependency upgrades during the build.** Pin everything in package.json.
7. **UK English in every UI string and comment.** Colour, organise, analyse, programme, behaviour.
8. **No em dashes.** Use commas, colons, semicolons.
9. **Push the cutting edge first, scale down on failure.** WebGPU first, full PBR first, full physics first.
10. **If a feature breaks twice, simplify. Do not pile complexity on broken foundations.**

## Architecture Overview

```
+-----------------+         +-----------------+         +-----------------+
|  Main Thread    |  RPC    |   CAD Worker    |         |  Render Loop    |
|  (React + UI)   | <-----> | (OpenCascade.js)|         |  (Three.js +    |
|                 | Comlink |                 |         |   WebGPU)       |
+-----------------+         +-----------------+         +-----------------+
        |                                                       ^
        v                                                       |
+-----------------+         +-----------------+                 |
|  Zustand Store  |         |  Physics Worker |---------------->|
|  (state graphs) | <-----> |  (Rapier3D)     |  per-frame      |
|                 |         |                 |  transforms     |
+-----------------+         +-----------------+                 |
```

**Why two workers:** OpenCascade.js does heavy boolean and feature operations infrequently but they take hundreds of milliseconds. Rapier runs every frame at 60 Hz. Keeping them in separate workers stops physics from stalling when a sketch is being extruded.

## State Model

```ts
// Conceptual shape, full Zod schemas in src/state/schemas.ts

type SketchPlane = 'XY' | 'XZ' | 'YZ' | { customId: string };

type SketchPrimitive =
  | { type: 'line'; start: [number, number]; end: [number, number] }
  | { type: 'arc'; centre: [number, number]; radius: number; startAngle: number; endAngle: number }
  | { type: 'circle'; centre: [number, number]; radius: number }
  | { type: 'rectangle'; corner: [number, number]; width: number; height: number };

type Sketch = {
  id: string;
  name: string;
  plane: SketchPlane;
  primitives: SketchPrimitive[];
};

type Feature =
  | { id: string; type: 'extrude'; sketchId: string; depthMm: number; symmetric: boolean }
  | { id: string; type: 'revolve'; sketchId: string; axis: 'X' | 'Y' | 'Z'; angleDeg: number }
  | { id: string; type: 'fillet'; targetEdges: string[]; radiusMm: number }
  | { id: string; type: 'chamfer'; targetEdges: string[]; sizeMm: number }
  | { id: string; type: 'hole'; targetFace: string; positionXY: [number, number]; diameterMm: number; depthMm: number }
  | { id: string; type: 'boolean'; operation: 'union' | 'subtract' | 'intersect'; targetIds: string[] };

type Part = {
  id: string;
  name: string;
  sketches: Sketch[];
  features: Feature[]; // ordered, regenerated top to bottom
  materialId: string;
  // computed at regen time
  meshHash?: string;
  volumeCm3?: number;
  massKg?: number;
};

type Mate =
  | { id: string; type: 'fixed'; partA: string; partB: string }
  | { id: string; type: 'revolute'; partA: string; partB: string; axisLocal: [number, number, number]; pivotLocal: [number, number, number]; motorSpeedRpm?: number }
  | { id: string; type: 'prismatic'; partA: string; partB: string; axisLocal: [number, number, number]; motorForceN?: number }
  | { id: string; type: 'spherical'; partA: string; partB: string; pivotLocal: [number, number, number] }
  | { id: string; type: 'planar'; partA: string; partB: string; planeLocal: [number, number, number, number] };

type Assembly = {
  id: string;
  name: string;
  parts: Part[];
  mates: Mate[];
  groundPartId: string; // pinned to world
};

type SimulationState = {
  running: boolean;
  timeStepMs: number;
  gravity: [number, number, number]; // default [0, -9.81, 0]
};
```

## Build Phases

Each phase is a single Replit Agent prompt. Andrew reviews each, then issues the next prompt. STOP gates between phases.

### Phase 0: Project Scaffold

Vite 8 + React 19 + TypeScript 6 + Tailwind 3.4. App shell with a left sidebar, top toolbar, central 3D canvas placeholder, right inspector panel. Dark mode by default, technical-drawing aesthetic. Replit Orange accent #FF6B1A. Empty Zustand store at src/state/store.ts. Acceptance: clean scaffold, two routes (Modeller, Simulator) with React.lazy, hot reload working.

### Phase 1: WebGPU Canvas + OpenCascade.js Worker

Three.js r184 WebGPU renderer mounted in central canvas. Grid floor with origin axes (X red, Y green, Z blue). Orbit camera. PBR environment lighting via pmrem from a default HDR. OpenCascade.js loaded in Web Worker, Comlink-wrapped, exposes a single test method that creates a 10mm cube and returns a tessellated mesh. Render the cube. Acceptance: cube visible on deployed URL in external Chrome, FPS >= 60, kernel init time logged to console.

### Phase 2: Sketch Mode (2D Plane Drawing)

Sketch mode toggle in toolbar. Selecting a default plane (XY, XZ, YZ) opens a 2D drawing overlay. Tools: line, arc (3-point), circle, rectangle. Click to place vertices, escape to finish. Snap to grid (1mm default), snap to existing endpoints. Sketch primitives stored in Zustand. Visual: thin orange lines on a faded grid plane. Exit sketch mode returns to 3D view, sketch persists as overlay geometry. Acceptance: draw a rectangle and a circle on the XY plane, exit sketch, primitives visible in scene.

### Phase 3: Extrude and Revolve Features

Feature tree panel on the right inspector. Select a sketch, click Extrude, enter depth in mm, choose direction or symmetric. OpenCascade worker performs MakeExtrude, returns mesh. Select a sketch, click Revolve, choose axis and angle. Worker performs MakeRevolve. Feature appears in tree, mesh appears in scene. Edit a feature parameter, scene regenerates. Acceptance: extrude a rectangle into a plate, revolve a circle into a cylinder, edit extrude depth and see live update.

### Phase 4: Modification Features (Fillet, Chamfer, Hole)

Edge selection in 3D view (hover highlight, click to select, multi-select with shift). Fillet feature: select edges, enter radius. Chamfer feature: select edges, enter size. Hole feature: select face, click position, enter diameter and depth. All operations dispatched to OpenCascade worker. Acceptance: fillet four edges of a plate, chamfer a corner, drill a hole through the plate.

### Phase 5: Boolean Operations

Multi-part support. Two existing parts can be combined: union, subtract, intersect. UI: select two parts, click boolean operation, choose which is the tool body for subtract. OpenCascade worker performs the operation, replaces the two parts with a single result, original tree preserved as history. Acceptance: union two cubes, subtract a cylinder from a plate, see clean B-rep geometry.

STOP GATE 1: At this point the modeller is functional. Andrew reviews, decides whether to extend modelling or move to assembly. Default path: move to assembly.

### Phase 6: Multi-Part Scene Management

Parts panel in left sidebar lists all parts with show/hide and rename. Each part has its own local origin and transform. Dragging a part in 3D moves it (gizmo with translate and rotate handles). Parts persist their feature trees independently. Save scene to localStorage, load from localStorage. Acceptance: create three parts, position them in 3D, save, reload, parts return to saved positions.

### Phase 7: Mate Joint Creation

Switch to Assembly mode (toolbar toggle). Mate creation tool: pick first geometry (face, edge, or axis), pick second geometry on another part, choose mate type. Supported mates: Fixed, Revolute (axis-axis), Prismatic (axis-axis with face constraint), Spherical (point-point), Planar (face-face). Mates rendered as small icons at the joint location. One part designated as ground (pinned to world), default first part placed. Acceptance: assemble a base plate, two posts, and a connecting rod with revolute joints, mates visible in scene.

### Phase 8: Rapier3D Physics Integration

Rapier3D loaded in dedicated Physics Worker. Switch to Simulator view: physics world built from current assembly. Each part becomes a rigid body with inertia tensor computed from OpenCascade volume properties. Each mate becomes a Rapier joint. Static colliders auto-generated from tessellated mesh (convex decomposition for non-convex parts via Rapier convex hull). Press Play: world steps at 60 Hz, transforms streamed back to main thread, Three.js meshes update. Acceptance: drop the four-bar linkage in, press play, parts fall under gravity until joints constrain them.

### Phase 9: Motor Actuation

Revolute mate inspector: optional motor speed in RPM, optional motor torque limit in Nm. Prismatic mate inspector: optional motor force in N. When simulation runs, motors apply continuous actuation. Play/Pause/Reset controls. Speed control (0.25x, 0.5x, 1x, 2x). Acceptance: four-bar linkage with one motor on the input crank cycles continuously when Play is pressed.

STOP GATE 2: The killer demo is now possible. Andrew records the screen capture for LinkedIn here. Phases 10 onwards are polish.

### Phase 10: Material Library and Mass Properties

Built-in materials: Aluminium 6061 (2.70 g/cm3), Steel 1018 (7.87 g/cm3), Brass C36000 (8.50 g/cm3), Titanium Grade 5 (4.43 g/cm3), Nylon 6 (1.14 g/cm3), PLA (1.25 g/cm3), ABS (1.04 g/cm3), Acrylic (1.18 g/cm3). Assigning a material updates part appearance (PBR settings) and recomputes mass for physics. Mass properties panel shows volume in cm3, mass in kg, centre of mass in mm. Acceptance: change a part from aluminium to steel, mass roughly triples, simulation behaviour visibly changes.

### Phase 11: STL Export

Export current part or whole assembly to STL. Binary STL format. OpenCascade tessellation drives the mesh. Download triggers immediately. Acceptance: export the four-bar linkage, open in any STL viewer, geometry intact.

### Phase 12: Polish and Performance

Selection bloom outline (Replit Orange). Grid floor fade with distance. Skybox or solid dark gradient (designer choice). Toolbar icons refined. Empty states for no-parts, no-sketch, no-assembly. Loading state for OpenCascade init. FPS counter in dev mode only. Performance pass: ensure 60 FPS with 10 moving parts. Final build, deploy, capture screen recording. Acceptance: deployed URL runs at 60 FPS on M4 Chrome, screen recording produced, ready to post.

## Visual Design Language

**Aesthetic direction:** technical-drawing-meets-cinema. Dark navy background (#0A0E1A), grid in slightly lighter navy (#141B2E), Replit Orange accent for selections and motors (#FF6B1A), Replit Off-White for primary text (#F5F5F5), muted blue-grey for secondary UI (#7A8599).

**Typography:** Display font JetBrains Mono for measurements, technical labels, and toolbar. Body font Inter for inspector panels and dialogues. Numbers should always be tabular figures.

**Motion:** Subtle hover lifts on cards, smooth 200ms easing for panel transitions, no decorative animation. The mechanism running is the animation.

**Iconography:** Lucide icons throughout. Custom mate icons (small SVG glyphs) for the four mate types: filled circle for Fixed, ring for Revolute, double-ended arrow for Prismatic, sphere for Spherical, parallel bars for Planar.

## Risk Register

| Risk | Likelihood | Impact | Mitigation |
|------|-----------|--------|------------|
| OpenCascade.js WASM init takes >15s on first load | Medium | High | Add proper loading screen with progress, cache WASM in service worker after first load |
| Rapier joints unstable at high motor speeds | Medium | Medium | Cap motor speed at 600 RPM, increase solver iterations |
| Convex decomposition fails on complex parts | Medium | Medium | Fall back to bounding box collider, log warning |
| WebGPU support drops on a Chrome update | Low | High | Pin Chrome version Andrew tests on, document |
| 24 hours runs out before Phase 9 | Medium | High | STOP GATE 1 lets us ship a static modeller as Plan B; STOP GATE 2 gives us the demo even if Phases 10 to 12 are skipped |

## Plan B Scope Cuts (in priority order)

If running short on time, cut in this order:
1. Phase 12 polish (ship the rougher build)
2. Phase 11 STL export
3. Phase 10 material library (use a single default material)
4. Phase 5 booleans (skip, ship without unions and cuts)
5. Phase 4 modification features (skip fillet and chamfer)

Phases 0 to 9 are the minimum for the killer demo. Everything else is bonus.
