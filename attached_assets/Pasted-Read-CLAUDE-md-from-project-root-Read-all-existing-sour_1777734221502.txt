NumericInput component:

Number input with up/down step arrows on the right.
Step button: hold to repeat at 100ms interval, accelerate to 50ms after 1s.
Min/max clamping: typing 5000 in a max-1000 field clamps to 1000 on blur.
Debounce: external onChange fires 200ms after the user stops typing or stepping.
Keyboard: arrow up/down steps by step, arrow up/down with shift steps by step * 10.
Display formatting: 1 decimal place for mm fields (10.0), zero decimals for angle/integer fields (360).

SegmentedControl component:

Three buttons in a row, equal width, JetBrains Mono uppercase labels.
Active button: solid Replit Orange background, dark navy text.
Inactive buttons: transparent background, off-white text, 1px border #1F2942.
Hover on inactive: background rgba(255, 107, 26, 0.12).

Live preview rendering pipeline:
When featureEditor is active and livePreview is on:

Watch featureEditor state for parameter changes via Zustand subscription.
On change, debounce 200ms.
Call previewFeature(featureEditor, sourceSketch, kernel) from featureRegen.ts. This returns a TessellatedMesh (or error).
Pass the mesh to PreviewMeshLayer which renders it as a single Mesh with the standard PBR material at 0.85 opacity (slight transparency to indicate it's a preview).
If the operation errors (wire not closed, self-intersection, etc), the preview mesh is hidden and the error message renders in the inspector under the parameters in red #FF6B6B.

Apply flow:

Inspector's Apply button calls applyFeatureEditor() from the store.
Store action either inserts a new feature into part.features or replaces an existing one.
PartMeshLayer subscribes to assembly.parts[].features changes, calls regeneratePart(part, kernel) for the affected part, replaces the mesh.
PreviewMeshLayer clears its preview mesh.
Inspector closes (featureEditor reset to null), feature tree highlights the newly applied feature.

Cancel flow:

Inspector's Cancel button calls cancelFeatureEditor().
PreviewMeshLayer clears its preview mesh.
Inspector closes.
The original part mesh (if editing an existing feature) remains unchanged.

PartMeshLayer:
Imperative manager hung off Scene.tsx. Maintains a Map<string, Mesh> keyed by partId. On every assembly change:

For each part in assembly.parts:

If the part has features: call regeneratePart(part, kernel), get TessellatedMesh, build BufferGeometry, update or create Mesh.
If the part has no features: hide its mesh.


For each Mesh in the map not in current assembly: dispose geometry, dispose material if not shared, remove from scene.

Use the same MeshStandardMaterial settings as Phase 1 test cube: colour #A0A8B5, metalness 0.4, roughness 0.5. Phase 10 will replace this with the material library.
Test cube removal:
The Phase 1 test cube creation in Scene.tsx (the call to kernel.createTestCube(10)) should be removed. The scene starts empty until parts are created via the modelling flow.
The kernel's createTestCube worker method can stay for diagnostic purposes (the runSelfTest from Split A uses a similar code path); just don't call it from Scene.tsx on mount.
Error handling:
Errors from the worker (returned as { ok: false, error: 'wire-not-closed' | ... }) need user-facing messages:

wire-not-closed: "Sketch must be closed. Connect all endpoints before extruding."
wire-empty: "Sketch is empty. Add primitives before extruding."
wire-multiple-closed: "Sketch contains multiple closed loops. Only one loop is supported in v1."
wire-mixed: "Sketch mixes open and closed primitives. All primitives must form a single closed loop."
self-intersection: "Revolve failed: sketch crosses the revolution axis."
occt-internal: "Operation failed. Try a simpler sketch or different parameters."

These render in the inspector under the parameter controls in #FF6B6B, with a small warning icon.

Note for Agent:
The 200ms debounce on live preview is critical. Without it, fast slider drags will queue up dozens of worker calls and the UI will feel laggy or freeze. Use a single timer that resets on each parameter change.
If two parts both have the same single-feature hash (e.g. two parts with identical 10mm cubes), they should share the cached mesh. Don't create separate MeshStandardMaterial instances per part either; one shared default material is enough until Phase 10.
BufferGeometry requires setAttribute('position', ...), setAttribute('normal', ...), and setIndex(...) with the right typed arrays from the TessellatedMesh. Don't forget geometry.computeBoundingSphere() or orbit controls may behave oddly with off-origin geometry.
