Topology enumeration approach:
OpenCascade.js exposes faces and edges via TopExp_Explorer. The kernel needs to:

After tessellating a shape, walk its edges with TopExp_Explorer(shape, TopAbs_EDGE) and assign a stable ID per edge.
Walk faces with TopExp_Explorer(shape, TopAbs_FACE) and assign a stable ID per face.
Stable IDs are derived from a canonical hash of the edge/face's underlying geometry (curve type, endpoint coordinates rounded to 0.001mm, surface type, centroid coordinates rounded to 0.001mm). This means edges and faces survive parameter edits in upstream features as long as their geometric identity is preserved.
For each edge, capture: type ('line' | 'circle' | 'arc' | 'spline' | 'other'), length in mm, midpoint in 3D, and the indices into the tessellated mesh that approximate this edge (a polyline of vertex indices).
For each face, capture: type ('plane' | 'cylinder' | 'cone' | 'sphere' | 'torus' | 'other'), area in mm², centroid in 3D, normal at centroid, and the triangle indices that belong to this face.
Return this metadata as part of the TessellatedMesh result, not as a separate worker call. Adding it to the existing return path avoids round-trips.

Picker selection states:
The 3D view supports four picking modes, controlled by the active inspector:

'idle' — no picking. This is the default outside Fillet/Chamfer/Hole inspectors.
'edges' — hover highlights edges, click selects. Multi-select with shift. Used by Fillet and Chamfer.
'faces' — hover highlights faces, click selects single face. Used by Hole (face on which to drill).
'point-on-face' — first click selects a face, second click places a 2D point on that face's plane. Used by Hole (placement).

Picking mode is stored in Zustand under featureEditor.pickingMode (Split B will populate this when Fillet, Chamfer, or Hole inspectors are active).

Implementation notes:

Edge highlights: per-edge Line2 from EdgeHighlightLayer. Rebuild on selection change. The polyline indices from the metadata go straight into a LineGeometry.
Face highlights: build a small overlay mesh from the face's triangle indices. Material is MeshBasicMaterial with transparent: true, opacity: 0.15 | 0.30, depthWrite: false, side: DoubleSide, polygonOffset: true, polygonOffsetFactor: -1. The polygon offset prevents z-fighting with the underlying part mesh.
Outline around selected face: extract the face's boundary edges (its loops), render as a Line2 group at line width 2, full opacity orange.

Raycasting:
Standard Three.js Raycaster against the part mesh. On hit:

For face mode: identify which face the hit triangle belongs to using a triangle-to-face lookup table built from faces[].triangles. The triangle index of the hit comes from the raycaster intersection.
For edge mode: edges are 1D, no triangles. Use a screen-space proximity test: project all edge polylines to screen space, find the edge whose nearest segment is closest to the cursor and within 8px. This is more reliable than raycasting against thin lines.

Build the triangle-to-face lookup once per mesh regen and cache it. Key by partId.

Edge proximity raycasting (the trickiest piece):
tsfunction nearestEdgeAtCursor(
  cursorScreen: [number, number],
  camera: Camera,
  rendererSize: [number, number],
  partMeshes: Array<{ partId: string; positions: Float32Array; edges: EdgeMetadata[] }>,
  thresholdPx: number = 8
): { partId: string; edgeId: string; distancePx: number } | null {
  // For each edge polyline:
  //   For each consecutive pair of vertex indices:
  //     Project both vertices to screen space using camera + rendererSize
  //     Compute distance from cursorScreen to the projected segment
  //   Track the minimum distance per edge
  // Return the edge with smallest min-distance, if under threshold
}
Project to screen space via standard NDC: vec.project(camera) gives NDC [-1, 1], then map to pixel coords. Cache projected positions per frame.

Note for Agent:
The stable hashing of edges and faces is what makes Fillet and Chamfer work across parameter edits. If you hash by vertex index in the tessellation, IDs change every regen and selection breaks. Hash by canonical geometry: line edges by their two endpoints rounded to 0.001mm, circular edges by centre + radius + axis + start/end angles rounded, faces by their underlying surface type + a canonical signature of the surface parameters.
The triangle-to-face lookup is critical for face raycasting. Build it as a Uint32Array parallel to the indices array, where each triangle index gets the face's ID-as-integer (or a face index into faces[]). Then the raycaster's triangle index lookup is constant time.
Polygon offset on the face overlay material is non-negotiable. Without it, you'll see flicker between the face overlay and the underlying part mesh on every camera move, even with depthWrite: false.
