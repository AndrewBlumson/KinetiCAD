The worker reconstructs the upstream shape from the feature chain (re-runs extrude or revolve), then applies fillet/chamfer/hole on top. Tessellate the result and return.
Why re-execute upstream rather than caching the OCCT shape:
OCCT shapes can't be transferred across the worker boundary (they're WASM heap pointers). The kernel keeps no persistent state between worker calls (each call is independent). Re-execution is the only safe path. The featureCache from Phase 3 still works because it caches the resulting TessellatedMesh, not the OCCT shape; cache hits avoid re-execution entirely.
Edge ID resolution inside the worker:
When the worker re-executes upstream and arrives at the modification step, it needs to find OCCT edges matching the stored edge IDs. The topology enumeration logic from Split A produces stable IDs from canonical geometry. Re-run the same enumeration on the freshly-built upstream shape, get a Map<edgeId, TopoDS_Edge>, look up each target edge by ID. If an ID can't be resolved, return { ok: false, error: 'edge-not-found', details: 'Edge ID xxx no longer exists after upstream change' }.
Same approach for face IDs in hole operations.
Fillet operation:
tsfunction fillet(shape: TopoDS_Shape, edgeMap: Map<string, TopoDS_Edge>, edgeIds: string[], radiusMm: number): TopoDS_Shape {
  const filletMaker = new oc.BRepFilletAPI_MakeFillet(shape);
  for (const id of edgeIds) {
    const edge = edgeMap.get(id);
    if (!edge) throw new Error(`Edge not found: ${id}`);
    filletMaker.Add_2(radiusMm, edge);  // single-radius signature
  }
  filletMaker.Build();
  if (!filletMaker.IsDone()) throw new Error('Fillet failed');
  const result = filletMaker.Shape();
  filletMaker.delete();
  return result;
}
Common failure modes:

Radius too large for the edge geometry: surface as 'fillet-radius-too-large'.
Adjacent fillets self-intersecting: surface as 'fillet-self-intersect'.

Chamfer operation:
tsfunction chamfer(shape: TopoDS_Shape, edgeMap: Map<string, TopoDS_Edge>, edgeIds: string[], sizeMm: number): TopoDS_Shape {
  const chamferMaker = new oc.BRepFilletAPI_MakeChamfer(shape);
  for (const id of edgeIds) {
    const edge = edgeMap.get(id);
    if (!edge) throw new Error(`Edge not found: ${id}`);
    chamferMaker.Add_2(sizeMm, edge);   // equal-distance chamfer
  }
  chamferMaker.Build();
  if (!chamferMaker.IsDone()) throw new Error('Chamfer failed');
  const result = chamferMaker.Shape();
  chamferMaker.delete();
  return result;
}
Hole operation:
tsfunction hole(
  shape: TopoDS_Shape,
  faceMap: Map<string, TopoDS_Face>,
  targetFaceId: string,
  positionUV: [number, number],
  diameterMm: number,
  depthMm: number
): TopoDS_Shape {
  const face = faceMap.get(targetFaceId);
  if (!face) throw new Error(`Face not found: ${targetFaceId}`);

  // 1. Get the face's surface and evaluate at UV to get 3D point
  const surface = oc.BRep_Tool.Surface_2(face);
  const point3d = new oc.gp_Pnt_1();
  surface.get().D0(positionUV[0], positionUV[1], point3d);

  // 2. Get face normal at that point (the cut direction is -normal, into the body)
  const normal = computeFaceNormalAt(face, positionUV);

  // 3. If depthMm === 0 (through-all), use a depth that's larger than the body's bounding box
  const cutDepth = depthMm === 0 ? boundingBoxDiagonal(shape) * 2 : depthMm;

  // 4. Build a cylinder primitive at point3d, oriented along -normal, radius diameter/2, height cutDepth
  const axis = new oc.gp_Ax2_3(point3d, new oc.gp_Dir_2(-normal[0], -normal[1], -normal[2]));
  const cylinder = new oc.BRepPrimAPI_MakeCylinder_3(axis, diameterMm / 2, cutDepth).Shape();

  // 5. Boolean subtract the cylinder from the body
  const cutMaker = new oc.BRepAlgoAPI_Cut_3(shape, cylinder);
  cutMaker.Build();
  if (!cutMaker.IsDone()) throw new Error('Hole boolean failed');
  const result = cutMaker.Shape();

  // 6. Cleanup intermediates
  point3d.delete();
  axis.delete();
  cylinder.delete();
  cutMaker.delete();

  return result;
}
The computeFaceNormalAt helper uses BRepAdaptor_Surface and the face's UV derivatives. If the face is reversed (face.Orientation() === TopAbs_REVERSED), flip the normal.
Through-all depth: when depthMm === 0, drill through the entire body. Implement by setting cylinder length to twice the body's bounding box diagonal, then booleaning.
Boolean tolerance: OCCT's default boolean tolerance can fail on tight intersections. If BRepAlgoAPI_Cut.IsDone() returns false, retry with cutMaker.SetFuzzyValue(0.001) (1 micron). If that also fails, surface as 'boolean-failed'.

Through-all hole depth (depthMm === 0) is the most likely place a unit conversion bug hides. The cylinder length needs to be in the same units as the body (mm). If you compute "twice the bounding box diagonal" in metres and use it as mm, the hole will be 1/1000 the depth needed. Sanity-check: a 10mm cube's through-all hole should use a cylinder of about 35mm length, not 0.035mm.
BRepAlgoAPI_Cut's SetFuzzyValue is a useful escape hatch for tight tolerance failures, but don't apply it by default. Default tolerance produces cleaner topology; fuzzy tolerance can leave tiny faces. Use 0.001 (1 micron) as a fallback only.
