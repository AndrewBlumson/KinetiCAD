OpenCascade requires a closed wire for extrude and revolve operations. The sketch primitives need to be converted to OCCT topology in the right order, with shared endpoints connected as shared vertices.Approach:
Walk the sketch primitives and build a graph of endpoints. Two endpoints within 0.001mm are treated as the same vertex.
Order the primitives end-to-end so each primitive's end vertex is the next primitive's start vertex.
For each primitive, build an OCCT edge:

Line: BRepBuilderAPI_MakeEdge(gp_Pnt(start), gp_Pnt(end))
Circle: BRepBuilderAPI_MakeEdge(gp_Circ(centre, normal, radius)) produces a closed circular edge, on its own forms a closed wire
Arc: BRepBuilderAPI_MakeEdge(gp_Circ(centre, normal, radius), startAngle, endAngle)
Rectangle: emit four line edges in order (corner, corner+width, corner+width+height, corner+height, back to corner)


Compose the edges into a BRepBuilderAPI_MakeWire. If any edge does not connect, log the gap and reject.
Verify the wire is closed via wire.Closed(). If not closed, return an error. The user must finish their sketch before extruding.
Return the TopoDS_Wire. The caller (extrude.ts or revolve.ts) is responsible for .delete() on the wire and all intermediate vertices/edges.
Memory hygiene: every OCCT object created (gp_Pnt, gp_Circ, edge, wire, builder) must be .delete()'d after use.   // 1. Make a face from the wire: BRepBuilderAPI_MakeFace(wire)
  // 2. Compute extrude vector based on plane normal and direction:
  //    - Forward: +normal * depth
  //    - Backward: -normal * depth
  //    - Symmetric: translate face by -normal * (depth/2), then extrude +normal * depth
  // 3. Use BRepPrimAPI_MakePrism(face, gp_Vec(extrudeVector))
  // 4. Return the resulting solid
  // 5. Delete face, builder, intermediate vec
}Plane normals: XY plane normal is +Z, XZ is +Y, YZ is +X.Revolve operation (revolve.ts):tsfunction revolve(wire: TopoDS_Wire, axis: 'X' | 'Y' | 'Z', angleDeg: number): TopoDS_Shape {

Note for Agent:The wire-from-sketch logic is the part most likely to fail. Sketches that look closed visually can fail OCCT's wire.Closed() check if endpoints are not exactly coincident. The 0.001mm tolerance on vertex merging should handle the common case (snap-to-endpoint produces matching coords), but if a sketch was drawn without snap, gaps may exist.
