Tool sub-state (per tool):

Line: 'awaiting-start' | 'awaiting-end'. After commit, awaiting-start resumes with the just-committed endpoint as the next start (chain mode). Escape or re-click Line breaks the chain.
Rectangle: 'awaiting-first-corner' | 'awaiting-second-corner'. After commit, returns to awaiting-first-corner.
Circle: 'awaiting-centre' | 'awaiting-radius'. After commit, returns to awaiting-centre.
Arc: 'awaiting-start' | 'awaiting-end' | 'awaiting-mid'. After commit, returns to awaiting-start.

Snap engine spec:

Input: cursor position in world coords (already projected onto the sketch plane), all committed primitives, modifier keys.
Output: SnapResult { position: [u, v]; type: 'grid' | 'endpoint' | 'midpoint' | 'free'; sourcePrimitiveId?: string }.
Priority: endpoint > midpoint > grid. Free (no snap) only when Alt is held.
Endpoint and midpoint radius is 5px in screen space, which translates to a world-space radius dependent on camera zoom. Compute the radius per call from the active camera.
Grid snap step: 1mm. Round both u and v to nearest integer.
Snap markers render at the snap position whenever a tool is active and cursor is within the sketch plane.

Cursor projection:

Raycast from camera through cursor, intersect with the active sketch plane (it's already at world Z=0, Y=0, or X=0 depending on plane).
Convert the intersection point to plane-local 2D coordinates using worldToPlane from src/sketch/plane.ts.
Pass through snap engine before use.
If raycast misses the plane (cursor outside the 400mm sketch plane bounds), tools should still work but no snap marker is shown beyond grid.

In-flight rubber-band rendering:

Updated every mousemove via the existing setAnimationLoop frame.
Line: dashed orange line from start to current snap position.
Rectangle: dashed orange rectangle outline (4 sides).
Circle: dashed orange circle, segments=64.
Arc: dashed orange arc through start, current cursor (or fixed end after second click), with the third-point preview between.
All in-flight rendering uses Line2 with LineMaterial configured for dashed pattern: dashed: true, dashSize: 0.5, gapSize: 0.3, dashOffset: 0. Line width 1.5, opacity 0.7, colour #FF6B1A.
LineMaterial requires resolution set to canvas size in pixels and updated on resize.

Committed primitive rendering:

Solid orange #FF6B1A, line width 2.0, opacity 1.0, no dashes.
Same Line2 pipeline.
One Object3D per primitive, parented to a committedSketchGroup that lives inside the sketch overlay.
When the sketch session is finished (Phase 2 Split A wired this), the group should be re-parented to a persistent layer that renders even from 3D view at 0.6 opacity.

Persistent overlay (post-finish):

After finishSketch, the committed primitives should remain visible from any camera angle as a thin orange overlay.
Implementation: keep the committed sketch group, swap its material opacity from 1.0 to 0.6 and line width from 2.0 to 1.5.
One persistent group per finished sketch in the assembly.
Persistent groups added to a top-level scene group called finishedSketchesLayer.
Reload from localStorage rebuilds these groups from the persisted Sketch primitives.

Snap markers (visual):

Grid snap: solid orange filled circle, radius 0.4mm world-space.
Endpoint snap: solid orange diamond (rotated square), 0.8mm.
Midpoint snap: solid orange triangle, 0.8mm.
All implemented as small Mesh objects with MeshBasicMaterial (no lighting, ignored by depth via depthTest: false, renderOrder: 999) so they always appear on top.
Only one marker visible at a time, positioned at the active snap result.

Coordinate label:

A small DOM overlay (not a Three.js sprite, simpler to render text crisply) follows the cursor offset by (12px, -12px).
Format: (15, 25) with JetBrains Mono, white text, dark navy background #0A0E1A at 0.85 opacity, 4px padding, 1mm precision.
Visible only when a tool is active and cursor is within the sketch plane.

Crosshair cursor:

Replaces the system cursor inside the canvas while a tool is active.
Implementation: thin orange crosshair, two 16px lines, 1px wide, centred on cursor.
Use a CSS overlay element, not cursor: url(...), so we have crisp anti-aliased rendering and can match the Replit Orange exactly.
Hide when cursor is over UI elements (toolbar, inspector). Restore system cursor in those zones.

Mouse event wiring:

Mousedown on canvas while a tool is active: fire the tool's click handler with the snapped position.
Mousemove on canvas: update in-flight primitive's "current" point, update snap result, update snap marker, update coordinate label.
Right-click or Escape: cancel in-flight primitive (but stay in the same tool, ready for a new first click).
Click and drag should be ignored as separate events: this is click-place-click, not drag-to-draw.

Arc geometry math (this is the trickiest single piece):
Given three points P1 (start), P2 (end), P3 (point on arc):
ts// In src/sketch/arcGeometry.ts
export function arcFromThreePoints(
  p1: [number, number],
  p2: [number, number],
  p3: [number, number]
): { centre: [number, number]; radius: number; startAngle: number; endAngle: number } | null {
  // Compute perpendicular bisector of P1-P2 and P2-P3
  // Their intersection is the circle centre
  // Return null if the three points are collinear (no arc possible)
  const ax = p1[0], ay = p1[1];
  const bx = p2[0], by = p2[1];
  const cx = p3[0], cy = p3[1];

  const d = 2 * (ax * (by - cy) + bx * (cy - ay) + cx * (ay - by));
  if (Math.abs(d) < 1e-9) return null; // collinear

  const ux = ((ax * ax + ay * ay) * (by - cy) + (bx * bx + by * by) * (cy - ay) + (cx * cx + cy * cy) * (ay - by)) / d;
  const uy = ((ax * ax + ay * ay) * (cx - bx) + (bx * bx + by * by) * (ax - cx) + (cx * cx + cy * cy) * (bx - ax)) / d;

  const centre: [number, number] = [ux, uy];
  const radius = Math.hypot(ax - ux, ay - uy);

  // Angles
  const startAngle = Math.atan2(ay - uy, ax - ux);
  const endAngle = Math.atan2(by - uy, bx - ux);

  // Determine sweep direction: does P3 lie on the short arc or long arc from P1 to P2?
  // Compute angle of P3, check if it's between startAngle and endAngle going counter-clockwise.
  // If not, flip endAngle (sweep the long way round).
  // Implementation note: normalise all angles to [0, 2*PI) and check ordering.

  return { centre, radius, startAngle, endAngle };
}
Test this by drawing three different arcs that should sweep clockwise vs counter-clockwise. The sweep direction is the part most likely to come out wrong.

Note for Agent:
If Line2 rendering shows invisible or 1px-thick lines, the cause is almost always one of:

LineMaterial.resolution not set (must be (canvasWidth, canvasHeight))
Resolution not updated on resize
LineGeometry positions array not flat [x, y, z, x, y, z, ...]
Missing material.dashedSize and material.gapSize for dashed lines
Forgetting geometry.computeLineDistances() for dashed lines
